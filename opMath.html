Linear / Matmul 5 operators
Dense (Linear) All networks
Standard
y = Wx + b, W ∈ ℝ^(out×in)
2–10×
wall-time
▼
Base formula
y = W · x + b W ∈ ℝ^(out×in), x ∈ ℝ^in
Optimized — cuBLAS GEMM (tiled)
C[i,j] += Σ_k A[i,k] · B[k,j] Tile size: 128×128 register tiles Shared mem: avoids re-reading HBM per tile
Most ML compute reduces to GEMM. The key insight: load a tile of A and B into fast shared SRAM, compute the partial dot products, then accumulate — minimizing slow HBM round-trips.
Low-rank approx LoRA / SVD
Standard

LoRA decomposition
W ≈ B · A B ∈ ℝ^(m×r), A ∈ ℝ^(r×n) r ≪ min(m, n) Params: (m+n)·r vs m·n
LoRA freezes W and trains only ΔW = BA. At r=8, a 4096×4096 layer goes from 16.7M → 65K trainable parameters — a 256× reduction. Merged at inference: W' = W + BA, zero overhead.

INT8 quantized (per-channel scale)

Attention 6 operators
Scaled dot-product attention Transformers
Standard
softmax(QKᵀ/√d_k)V — O(N²) memory
20×
memory
▼
Standard (materializes N×N matrix)
S = QKᵀ / √d_k ∈ ℝ^(N×N) ← to HBM P = softmax(S) ← to HBM O = PV ← from HBM Memory: O(N²) e.g. 32k² = 4GB
FlashAttention-2 (online softmax)
Tile Q,K,V into SRAM blocks; no HBM write for S Online running stats per tile: m_i = max(m_{i-1}, rowmax(S_tile)) l_i = e^(m_{i-1}-m_i)·l_{i-1} + rowsum(e^(S-m_i)) O_i = diag(l_{i-1}/l_i)·O_{i-1} + e^(S-m_i)·V Final: rescale O by 1/l_N Memory: O(N) — N×N never written to HBM
The breakthrough: softmax is normally a two-pass algorithm (first pass for max, second for exp/sum). FlashAttention fuses them with a numerically stable running correction, so attention tiles can compute in-register without materializing the N×N matrix in HBM.

MHA → MQA → GQA
MHA: h_Q = h_K = h_V = h MQA: h_K = h_V = 1 (extreme) GQA: h_K = h_V = g, 1 ≤ g ≤ h

KV cache is the decode bottleneck at scale — every generated token reads all past K,V. GQA (used in LLaMA 3, Gemma, Mistral) cuts cache size by h/g with minimal quality degradation vs MHA.
Linear attention Long sequences
Standard

O(N)
complexity
▼
Standard softmax attention
Attn(Q,K,V) = softmax(QKᵀ/√d)V Cost: O(N²d) — quadratic in seq length
Kernel / linear attention (associative trick)
Replace softmax with kernel φ(·): Attn(Q,K,V)_i = φ(q_i)ᵀ(Σ_j φ(k_j)v_jᵀ) / φ(q_i)ᵀΣ_j φ(k_j) Matrix assoc: compute (φ(K)ᵀV) first → ℝ^(d×d) Cost: O(Nd²) vs O(N²d) φ options: ELU+1, random Fourier features, FAVOR+
The key algebraic trick: (AB)C ≠ A(BC) in general, but if softmax is replaced by a decomposable kernel, right-to-left evaluation is valid and reduces complexity from O(N²) to O(N). Used in Performer, RetNet, Mamba-style models.
RoPE encoding Position
Standard
Sinusoidal: h += PE[pos]
free

▼
Sinusoidal (additive)
PE[pos, 2i] = sin(pos / 10000^(2i/d)) PE[pos, 2i+1] = cos(pos / 10000^(2i/d)) h = token_embed + PE[pos] ← adds to residual
RoPE (rotary, multiplicative)
For each pair (q_{2i}, q_{2i+1}) at position m: q_m^rot = R_m · q_m R_m = block-diag of 2D rotation matrices: [cos(mθ_i) -sin(mθ_i)] θ_i = 10000^(-2i/d) [sin(mθ_i) cos(mθ_i)] Relative position in dot product: q_m^rot · k_n^rot = f(q_m, k_n, m-n)
RoPE encodes relative position directly into the attention score (q·k), not the residual stream. The dot product between two rotated vectors depends only on their relative offset m−n — enabling seamless length extrapolation. Used in LLaMA, PaLM, Gemma, Falcon.
Normalization 4 operators
Batch norm CNNs
Standard
x̂ = (x−μ_B)/√(σ²_B+ε), y = γx̂+β
2×
mem bandwidth
▼
3-pass naive
Pass 1: μ_B = (1/m) Σ x_i Pass 2: σ²_B = (1/m) Σ(x_i − μ_B)² Pass 3: y_i = γ·(x_i−μ_B)/√(σ²+ε) + β 3 full reads of x from HBM
Fused BN kernel (1-pass statistics)
Welford online algorithm: count, mean, M2 updated per element var = M2 / count (no reread needed) Normalize + scale in same kernel. Inference: fold γ,β into next layer's W,b: W' = W·γ/σ, b' = b + W·(−μγ/σ+β)
RMSNorm LLaMA, Gemma
LayerNorm
LN(x) = γ⊙(x−μ)/√(σ²+ε) + β
15%
faster than LN
▼
LayerNorm (full)
μ = (1/H) Σ x_i σ² = (1/H) Σ (x_i − μ)² LN(x) = γ ⊙ (x−μ)/√(σ²+ε) + β 2 stats to compute, 2 params γ,β
RMSNorm — drop mean subtraction
RMS(x) = √(1/H · Σ x_i²) RMSNorm(x) = γ ⊙ (x / RMS(x)) — no β (bias), no mean subtraction Fused warp-level reduction: __shfl_down_sync for Σ x_i² Single-pass, ~15% faster vs LN
Mean subtraction in LayerNorm is "re-centering" — hypothesis: this is redundant if the next layer has a bias. RMSNorm drops it, reducing the reduction kernel from 2 passes to 1. Empirically matches LN quality while being faster.

Normalization axis comparison
BatchNorm: normalize over (N, H, W) per C → needs large N LayerNorm: normalize over (C, H, W) per N InstanceNorm: normalize over (H, W) per (N,C) GroupNorm: normalize over C/G chans per (N, g)
GroupNorm math
Divide C channels into G groups. For group g in sample n: μ_{n,g} = mean(x[n, gC/G : (g+1)C/G, :, :]) σ²_{n,g} = var(same slice) y = γ·(x−μ)/√(σ²+ε) + β Typical: G=32, C≥32. G=1 → LayerNorm. G=C → InstanceNorm.
Spectral norm GANs
Standard

O(mn)
vs O(mn²) SVD
▼
Exact SVD
σ_max(W) = top singular value via full SVD Cost: O(m·n·min(m,n)) — expensive per step
Power iteration (1-step approx)
Maintain unit vectors û ∈ ℝ^m, v̂ ∈ ℝ^n Each forward pass: ṽ = Wᵀû / ‖Wᵀû‖ ũ = Wṽ / ‖Wṽ‖ σ̂ = ũᵀWv̂ SN(W) = W / σ̂ Update û←ũ, v̂←ṽ for next step. Cost: O(mn) — 2 matvecs only
One step of power iteration is sufficient in practice because W changes slowly during training. The running û, v̂ vectors "track" the top singular direction across steps, so each step only needs a small correction.

SwiGLU LLaMA, PaLM
Standard
Swish(xW+b) ⊙ (xV+c)

▼
Standard FFN

InfoNCE

Standard CE vs Focal
CE: L = −log(p_t) Focal: L = −(1−p_t)^γ · log(p_t) γ=0: reduces to CE γ=2: easy examples (p_t≈1) downweighted by (1−p_t)² ≈ 0

Optimizers 5 operators
AdamW Transformers, LLMs
Standard Adam
θ ← θ − η·m̂/(√v̂+ε)
decoupled
weight decay
▼
Adam (L2 weight decay as gradient)
g_t = ∇L + λθ ← decay in gradient m ← β₁m + (1-β₁)g_t v ← β₂v + (1-β₂)g_t² m̂ = m/(1-β₁ᵗ), v̂ = v/(1-β₂ᵗ) θ ← θ − η·m̂/(√v̂+ε) ← adaptive LR on λθ too
AdamW (decoupled decay)
g_t = ∇L ← decay NOT in gradient m ← β₁m + (1-β₁)g_t ← only loss gradient v ← β₂v + (1-β₂)g_t² θ ← (1−λη)θ − η·m̂/(√v̂+ε) ↑ fixed-rate decay, not adaptive-scaled 8-bit AdamW: m,v stored in INT8 w/ dynamic quant → 75% optimizer state memory saved
Adam's adaptive learning rates inadvertently scale down the weight decay term — large-gradient parameters see effectively weaker decay. AdamW fixes this by applying decay directly to θ at a fixed rate, independent of the adaptive scaling. This matters significantly for transformers.

4×

▼

v ∈ ℝ^(m×n) for W ∈ ℝ^(m×n) Cost: same memory as the parameter itself

B=32k

▼
Adam at large batch
Global η: tiny for stability → slow Linear LR scaling: η ∝ B causes divergence Problem: different layers have very different gradient magnitudes
LAMB: layer-wise trust ratio
r_l ← Adam update direction for layer l η_l = φ(‖θ_l‖) / ‖r_l‖ ← trust ratio φ: clip function (e.g. φ(z) = min(z, 10)) θ_l ← θ_l − η · η_l · r_l Each layer gets its own effective LR. Intuition: take a step proportional to current weight magnitude, direction from Adam.
LAMB enabled BERT pre-training in 76 minutes (vs days) by scaling to batch size 32,768. The key insight: normalize the update by layer norm and weight norm separately, removing sensitivity to layer-specific gradient scales.
Gradient checkpointing Memory
Standard
Store all activations: O(N·L) memory
O(√N)
memory
▼
Standard backprop
Store all layer activations during forward Memory: O(N·L) (N tokens, L layers) Fast backward: all acts immediately available
Selective recompute (√N strategy)
Store only every √L layer's activation During backward: recompute intermediate acts from nearest checkpoint. Memory: O(√L · N) Compute: +33% FLOPs (each layer computed ~1.33×) PyTorch: torch.utils.checkpoint.checkpoint() Selective: wrap only expensive layers (attn, FFN)
For a 100-layer model: standard stores 100 activation tensors; √N checkpointing stores ~10 and recomputes up to 10 layers per backward step. Particularly valuable for long-sequence training where activation memory dominates.
Mixed precision training AMP
Full FP32
All ops in FP32: 4 bytes/param
2–3×
speed + 2× mem
▼
Full FP32 training
Params, grads, optimizer states: FP32 4 bytes/element for everything cuDNN: no Tensor Cores for FP32 GEMM
AMP: automatic mixed precision
Weights: FP32 master copy (for stable updates) Forward + backward: FP16/BF16 (2 bytes) → 2× memory, 2-3× faster (Tensor Cores) Optimizer states: FP32 (precision critical) Loss scaling: multiply loss by S before backward → prevents FP16 underflow (grads too small) unscale before optimizer step: grad /= S BF16: same range as FP32, less precision → no loss scaling needed (same exponent bits)
BF16 (Brain Float 16) uses 8 exponent bits (same as FP32) vs FP16's 5 — this means BF16 handles the same value range as FP32 and doesn't need loss scaling. Modern LLM training (GPT-4, LLaMA) uses BF16 throughout with FP32 optimizer states.
